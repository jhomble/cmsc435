Usage

Usage should be mostly obvious but a few important notes:

1) A user cannot click on an elevator button unless it has already been requested. If you can't click the buttons, check that you first requested the elevator.

2) Average Passenger Wait Time only accounts for time waiting for an elevator to come after a floor button press. It will not be affected by time spent in an elevator.

3) It could take almost a minute for an elevator to answer a call depending on the current volume of traffic. This is not a bug, just how the AVW elevators really work.

4) Clicking on a floor button multiple times will increase the number of people waiting for the elevator in that direction (Yes, we know that isn't how the elevators really work but it is much easier for the user to enter).

How it Works

This simulator uses an HTML front-end utilizing the AngularJS framework to communicate with the JS logic.

Assumptions

1) Elevators in AVW use the FS algorithm described in "Algorithms."

2) Users actually exit the elevators at the floor they request (hence the button freeze).
